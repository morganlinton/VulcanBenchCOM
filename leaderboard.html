Every model configuration measured on VulcanBench, ranked: five models, nineteen configurations, five reports.

How to read this Rankings are within each report: every report holds one suite fixed and runs each model against the identical tasks. Suites differ between reports in size and difficulty, so scores across reports are not comparable: a higher percentage on an easier suite does not beat a lower one on a harder suite. When binary pass rates tie, the tie is broken by whatever still separates the models, a harder partial-credit tier, or economics: cost and cost per solved task.

Reproduce it

Every number here comes from the open-source harness, computed from full run traces: nothing is hand-entered. Regenerate any row with vulcanbench leaderboard, or reproduce a whole report from scratch: a two-model run costs about ten dollars. The grading underneath is deterministic hidden tests, spelled out in the methodology.
